# Task 02: Core Abstractions

## Goal

Identify the central abstractions that organize the pinned Opencode version before tracing detailed flows.

## Input Scope

- shared interfaces and base types
- core domain models
- registries, adapters, and service boundaries
- cross-cutting names such as agent, provider, tool, session, runtime, message, event

## Must Read

- `docs/{en,zh}/repo-map.md`
- `docs/{en,zh}/architecture.md`
- `scripts/symbol-grep.sh`

## Expected Output

- `reports/run-YYYY-MM-DD/core-abstractions.md`
- updates to `docs/{en,zh}/architecture.md`

## Completion Standard

- the main abstraction families are listed,
- representative files and symbols are identified,
- and each abstraction is tied to its role in the system.
