# Opencode 分析工作台设计

> 这个文件记录了用于初始化分析工作台的首版仓库设计。

## 目标

创建一个对 agent 友好的仓库，用来支持对某个固定版本的 Opencode 做反复、深入的分析，从仓库盘点一直推进到具体代码级文档。

## 选定方案

把仓库本身当成分析 harness：

- `AGENTS.md` 定义运行规则。
- `docs/` 存放长期知识。
- `tasks/` 把大分析目标拆成有边界的小任务。
- `templates/` 稳定输出质量。
- `reports/` 隔离临时发现。
- `scripts/` 减少重复的手工检查工作。
- `workspace/source/` 存放被固定版本的 Opencode 源码检出。

## 被否决的替代方案

### 仅靠 prompt 的方案

被否决，因为长期知识会停留在聊天上下文里，并在跨会话时逐渐漂移。

### 只有文档、没有任务和模板的方案

被否决，因为不同 agent 轮次的输出质量和执行顺序会出现过大波动。

## 初始交付物

- 仓库骨架
- agent 运行说明
- 版本元数据文件
- 基线文档
- 起始任务集
- 可复用模板
- 源码分析辅助脚本
