# 概览

## 目标

这个仓库用于对某个固定版本的 Opencode 做持续、可重复的深入分析。它是为 agent 执行而设计的工作台：任务有边界，模板稳定，文档可沉淀，并且从仓库盘点到调用链分析有清晰路径。

最终正式文档采用双语结构：

- 英文文档位于 `docs/en/`
- 中文文档位于 `docs/zh/`
- 运行过程中的阶段产物继续单语存放在 `reports/`

## 什么算完成

当这个工作台能够稳定支持下面这条工作链路时，就算真正可用：

1. 导入一个指定的 Opencode 版本，
2. 对源码做目录与文件盘点，
3. 识别入口点和模块边界，
4. 追踪关键运行时链路，
5. 发布带代码锚点的技术文档，
6. 记录尚未解决的问题，供后续轮次继续分析。

## 分析产物

- `blog/opencode-deep-dive.md`：面向读者的主文
- `architecture.md`：系统级架构总览
- `repo-map.md`：源码布局和关键入口
- `modules/*.md`：按模块拆分的分析文档
- `callflows/*.md`：关键执行链路文档
- `version-notes/*.md`：版本差异记录

推荐阅读顺序是先主文，再看附录作为展开证据：

- `blog/opencode-deep-dive.md`
- `architecture.md`
- `modules/*.md`
- `callflows/*.md`

## 非目标

- 维护通用的 Opencode 产品说明文档
- 替代原始源码仓库的 README
- 在长期文档中保存原始、嘈杂的草稿输出
